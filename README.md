# My Market AI

株価・指数・ETF・ニュースと自分のメモをまとめ、考えた理由や確認事項を時系列で振り返るための投資情報整理ノートです。HTML/CSS/JavaScriptだけで動作します。

## 目的

My Market AIは売買アプリではありません。市場情報を見て、そのとき気になった理由や確認したい事実を記録し、後から振り返るための個人用ノートです。

## 主な機能

- 任意の銘柄コード・表示名・種類を登録できるウォッチリスト
- 個別株・指数・ETFに対応した日本／米国の入力候補
- API接続可能な現在価格、前日比、直近30秒のCanvas簡易チャート
- 配信元・時刻・別タブリンク付きの仮ニュース
- ニュース見出しの共通テーマと重複を整理する事実ベースのAI要約表示
- 銘柄ごとのウォッチ理由、自由タグ、お気に入り
- 日付と分類を付けて何件でも保存できるメモ
- 「なぜ気になったか」「確認したこと」「次に確認したいこと」「決算」「配当」「ニュース」「その他」の分類
- 件数・最終メモ日・分類フィルター付きのメモ履歴
- JSONバックアップ・復元、UTF-8 BOM付きCSV出力
- 免責事項・プライバシーポリシー・利用規約の案内ページ
- 補助的な価格・変動率アラート
- localStorageによる登録対象とメモ履歴の保存
- ライト／ダークモード、PC 2列／スマートフォン1列表示

## 既存データの移行

旧Stock Alert Memoで保存した登録対象はそのまま読み込まれます。旧形式の単一メモは、初回読み込み時に日付付きの「その他」メモとして履歴へ移行されます。既存の旧分類名が付いた履歴は書き換えず、分類フィルターの候補へ追加して表示します。

## 使い方

`index.html` をブラウザで開くだけで動作します。ビルド処理と外部ライブラリは不要です。APIキーを設定しない場合は、価格・チャート・ニュース・AI要約がサンプル用データで動作します。

## 現在のデータモード

現在は `script.js` 冒頭の `DATA_MODE = 'demo'` で動作します。この状態では外部APIへの通信を行わず、画面上には「サンプル表示」と表示されます。

`DATA_MODE = 'api'` に変更しても `API_KEY` が空欄なら、自動的にデモモードへ戻ります。

## Twelve Dataの設定

現在価格は[Twelve DataのLatest Price API](https://twelvedata.com/docs#price)、チャートは[Time Series API](https://twelvedata.com/docs#time-series)へ接続できます。

1. Twelve Dataでアカウントを作成し、ダッシュボードからAPIキーを取得します。
2. `script.js` 冒頭の `API_KEY = ''` の空文字部分へ、ローカルで試すキーを入力します。
3. `DATA_MODE = 'demo'` を `DATA_MODE = 'api'` に変更します。

API価格に対応している仮候補は `AAPL`、`SPY`、`VOO`、`QQQ` です。それ以外はAPIモードでもデモ価格を使います。API価格は無料枠の消費を抑えるため、初回取得後は60秒間隔で更新します。ニュースは引き続き仮データで、AI要約欄は取得済みのニュース見出しからルールベースで生成します。

API価格の取得に失敗したカードには「価格取得に失敗しました。時間をおいて再度お試しください。」と表示されます。失敗しても仮チャート、仮ニュース、メモ履歴は動作を続けます。

> **重要:** GitHub Pagesは静的サイトのため、ブラウザへ配信したAPIキーを秘密にできません。実際のAPIキーを公開リポジトリへコミットしないでください。この設定欄はローカルでの接続練習用です。公開運用時は、APIキーを環境変数に保管できるサーバーレス関数などの中継処理を利用してください。

## Stage7：チャート本物化

`DATA_MODE = 'api'` かつAPIキー設定済みの場合、チャートはTwelve Dataの `/time_series` から終値の時系列を取得します。

- 1日：5分足、78件
- 1週間：1時間足、35件
- 1か月：日足、30件

同じ銘柄・期間のAPI結果は5分間キャッシュし、期間ボタンの連打による不要なAPI消費を抑えます。APIキー未設定、通信失敗、APIエラー、終値データが2点未満の場合は、同じ銘柄・期間のサンプルチャートへ自動的に戻ります。フォールバック時はチャートに `SAMPLE FALLBACK` と表示します。

APIから返された系列とデモ系列は、どちらも `{ symbol, range, source, points }` の共通形式で `renderChart()` に渡されるため、既存の線・価格ラベル・余白デザインを維持します。

## Stage8：AI要約強化

固定文だったAI要約を、取得済みニュースのタイトルと種類ラベルから自動生成するルールベース要約へ変更しました。`generateAISummary(newsItems)` が次の順で3〜5行程度に整理します。

- URLまたは正規化したタイトルが同じ重複記事をまとめる
- 決算・業績、製品・サービス、経済指標、ETF、市場全体・指数、企業発表・事業動向の共通テーマを抽出する
- 記事数が多いテーマと、そのほかに含まれるテーマを事実ベースで記述する
- ニュースが2件未満の場合は「十分なニュースがありません」と表示する
- 投資判断や将来予測を目的としないことを要約内に明記する

要約生成はニュース一覧の描画後に独立した例外処理で実行します。要約処理でエラーが起きても、ニュースのリンク・配信元・時刻・種類表示には影響せず、要約欄だけが専用のエラーメッセージへ切り替わります。

将来AI APIへ接続する場合は、ニュース取得や画面描画を変更せず、`generateAISummary(newsItems)` の内部を安全な中継API呼び出しへ差し替える構成です。

## Stage8.5：ウォッチリスト自由追加

既存の登録フォームを拡張し、固定候補にない銘柄コードもウォッチリストへ追加できるようにしました。

- 銘柄コード、任意の表示名、種類、最初のメモを入力可能
- 種類は「個別株」「ETF」「指数」「その他」から選択
- 表示名が空欄の場合は候補データの名前、候補もない場合は銘柄コードを使用
- 銘柄コードは入力時と保存時に大文字へ統一
- 空欄・使用できない文字・大文字小文字を問わない重複登録を防止
- 追加直後に既存カードと同じUIへ反映し、`localStorage`へ保存
- 再読み込み後も登録対象、メモ履歴、種類などを維持
- 既存の削除ボタンで、追加した対象とそのメモ履歴を削除可能

候補外の銘柄も、デモ価格・デモチャート・種類と地域に応じたデモニュース・AI要約・メモを表示します。APIモードでは既存のAPI対応範囲を利用し、APIキー未設定やチャート取得失敗時はデモデータへ戻ります。価格取得に失敗した場合はカード内に短いエラーを表示し、ニュースとメモは動作を続けます。

保存データには今後の拡張用として、従来の `symbol`、`name`、`type`、`notes` に加え、`assetClass`、`memo`、`favorite`、`tags` を保持します。以前の保存データは読み込み時に不足項目を補うため、保存キーを変更せずに引き続き利用できます。

> **公開時の注意:** APIキーはフロントエンドのJavaScriptへ直書きせず、サーバーレス関数などの中継APIで環境変数として管理してください。

## Stage9：投資ノート強化

各カードへ、あとから振り返るための整理機能を追加しました。

- ウォッチ理由：自由入力した理由を銘柄ごとに保存
- タグ：自由入力のタグを最大12件まで追加し、カード上部にも表示
- お気に入り：カード右上の `☆ / ★` で切り替え、カード枠でも状態を識別
- メモ分類：「なぜ気になったか」「確認したこと」「次に確認したいこと」「決算」「配当」「ニュース」「その他」
- 履歴フィルター：分類ごとに絞り込み、該当件数と全件数を表示
- メモ統計：メモ総数と最終メモ日をカード内に表示
- 履歴表示：新しい順を維持し、履歴部分をスクロール可能な高さに制限

ウォッチ理由とタグ編集欄は折りたたみ式です。お気に入りと最大3件のタグはカード上部へ常時表示し、情報を増やしてもカードが縦に長くなりすぎない構成にしています。

保存データには `watchReason: ""`、`tags: []`、`favorite: false`、`notes: []` を保持します。古い保存データに項目がない場合は `normalizeStoredStock()` が初期値を補完するため、従来の登録対象・メモ・価格表示を維持したまま移行できます。フィルターは表示状態のみで、メモ本文や分類を書き換えません。

## Stage9.5：データ出力・バックアップ機能

登録フォームの上に折りたたみ式の「データ管理」を追加しました。バックアップファイルには個人的なメモが含まれるため、安全な場所へ保管してください。

### JSONバックアップ

`my-market-ai-backup-YYYYMMDD.json` として、次の内容を保存します。

- バックアップ形式のバージョンと出力日時
- ライト／ダークのテーマ設定
- ウォッチリスト、表示名、種類、ウォッチ理由、タグ、お気に入り
- 各銘柄のメモ履歴と既存表示に必要なデータ

### JSONから復元

新しいバックアップ形式に加え、配列だけの旧localStorage形式、`stocks` または `data` 配列を持つ旧形式も読み込めます。復元前に次の検証を行います。

- JSONとして解析できること
- ウォッチリスト配列と有効な銘柄コードがあること
- 同じ銘柄コードが重複していないこと
- ファイルサイズが5MB以下、登録対象が500件以下であること

検証に成功した場合だけ「現在の保存データを上書きします」という確認ダイアログを表示します。キャンセルまたは検証失敗時は既存データを変更しません。復元時は `normalizeStoredStock()` と `normalizeStoredNote()` が不足項目や古いメモ形式を補完し、保存後に価格・チャート・ニュース・AI要約を再描画します。

### CSV出力

メモ履歴を `my-market-ai-notes-YYYYMMDD.csv` として出力します。列は `symbol`、`name`、`type`、`favorite`、`tags`、`watchReason`、`noteDate`、`noteCategory`、`noteText` です。Excelで開きやすいようUTF-8 BOMとCRLF改行を使用し、タグは ` | ` 区切りで格納します。メモがない登録対象も、メモ列が空の1行として出力します。

## データ取得層

画面描画とデータ取得を分離するため、次の共通関数を設けています。

- `fetchMarketData(symbol)`：現在価格、前日比、通貨、取得時刻、データ提供元
- `generateDemoChartData(symbol, range)`：銘柄・期間別のデモ価格配列を生成
- `fetchTwelveDataChart(symbol, range)`：Twelve Dataから終値系列を取得
- `fetchChartData(symbol, range)`：API取得とデモフォールバックを統括
- `renderChart(symbol, chartData, card)`：取得済みデータをCanvasへ描画
- `generateDemoNewsData(symbol)`：対象の種類・地域に合わせた仮ニュースを生成
- `fetchNewsData(symbol)`：見出し、配信元、時刻、種類ラベル、URLを取得
- `renderNews(symbol, newsData, card)`：取得済みニュースをリンク付きで描画
- `generateAISummary(newsItems)`：ニュース見出しの重複と共通テーマを整理した事実ベース要約

現在価格とチャートは `DATA_MODE` に対応するデータ提供元を呼び出します。ニュースは明示的にデモ提供元を使用します。画面側は同じ形式の戻り値を利用するため、将来の差し替えでも表示処理を変更する必要がありません。

## 将来のAPI連携

現在は `dataProviders.api.fetchMarketData` と `dataProviders.api.fetchChartData` をTwelve Dataへ接続しています。ニュースはデモデータで、AI要約はそのニュース見出しからブラウザ内で生成します。チャートは `fetchTwelveDataChart / generateDemoChartData → fetchChartData → renderChart`、ニュースと要約は `generateDemoNewsData → fetchNewsData → renderNews → generateAISummary` の順に処理され、取得・描画・要約が分離されています。

次の接続候補はニュースAPIです。`dataProviders.api.fetchNewsData` を実装して `API_FEATURES.news` を `true` にする設計です。ニュース取得だけ失敗した場合は専用メッセージを表示し、価格・チャート・メモは動作を続けます。

検索候補は `stockDataProvider` の `search()` と `getBySymbol()` を検索APIへ置き換えられます。各候補は種類、地域、市場、更新方式を持ちます。投資信託向けには `assetTypeSettings` に日次更新方式を定義していますが、現時点では候補データに含めていません。

APIキーをブラウザ側へ直接埋め込まず、安全な中継処理を利用してください。
